About LockedIn AI

LockedIn AI is the #1 real-time AI interview and meeting copilot, trusted by over 1 million users worldwide. We are building the most advanced AI-powered career preparation platform that helps users perform better in interviews, coding assessments, and professional communication scenarios.

Our platform delivers real-time intelligence during live interactions — helping users respond with clarity, confidence, and accuracy.

Role Overview

We are looking for a deeply technical LLM Developer to design, build, and optimize the large language model systems powering LockedIn AI’s real-time copilot.

This is a core engineering role where you will directly work on the models that listen, reason, and respond in real time for millions of users.

You will own the full lifecycle of LLM systems — from model integration and prompt architecture to fine-tuning, inference optimization, and production deployment.

Key Responsibilities
1. LLM Application Development
Build production-grade LLM features for real-time interview and meeting assistance
Design prompt chains, multi-turn conversation systems, and agentic workflows
Develop retrieval-augmented generation (RAG) systems for grounded responses
Build multi-model orchestration systems for optimal performance and cost efficiency
2. LLM Integration & Optimization
Integrate APIs from OpenAI, Anthropic, Google, and open-source models
Build low-latency inference pipelines for real-time AI responses
Optimize performance using streaming, caching, batching, and context compression
Implement token optimization strategies for cost and performance balance
3. Fine-Tuning & Model Customization
Fine-tune LLMs for domain-specific use cases (interviews, coding, communication)
Build and manage training datasets for model improvement
Run experiments across base models and fine-tuning strategies
Apply LoRA, QLoRA, RLHF, DPO, and adapter-based techniques
4. Prompt Engineering & System Design
Design and optimize system prompts for accuracy, tone, and relevance
Build prompt evaluation frameworks with measurable quality metrics
Maintain prompt versioning and A/B testing systems
Debug hallucinations, failures, and edge-case behaviors
5. Evaluation & Monitoring
Build LLM evaluation pipelines (automated + human + LLM-as-judge)
Monitor production metrics: latency, accuracy, hallucination rate, cost
Implement drift detection and quality regression tracking
Continuously improve response quality based on real user data
6. Safety & Responsible AI
Build guardrails and filters for safe LLM outputs
Test systems against prompt injection and adversarial attacks
Ensure privacy-first AI design and secure data handling
Improve fairness, bias reduction, and system reliability
